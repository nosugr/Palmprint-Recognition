# 掌纹识别门禁系统 · 课程设计计划书

> 生物特征识别课程 · 3人团队 · 7天周期 · 硬件 + 前端可视化

---

## 1. 项目概述

本项目面向生物特征识别课程，设计并实现一套基于掌纹识别的嵌入式门禁系统。系统以树莓派为核心控制单元，通过摄像头采集掌纹图像，运用 Gabor 竞争编码算法提取特征并完成匹配，最终驱动电磁门锁实现物理开闭控制，并提供完整的 Web 可视化管理界面。

### 项目亮点

- **硬件真实可演示**：小木框 + 电磁锁，识别成功后物理弹开，效果直观，适合答辩录像
- **NIR 补光提升图质**：近红外灯环改善采图质量，掌纹纹路清晰
- **完整 Web 管理端**：注册、验证、日志全流程可视化
- **专业性能评估**：给出 EER 曲线，量化系统性能指标

### 基本参数

| 项目 | 内容 |
|------|------|
| 核心目标 | 掌纹识别自动开锁 |
| 硬件平台 | 树莓派 4B + USB 摄像头 |
| 展示方式 | 实物演示台 + Web 前端 |
| 评估指标 | Top-1 准确率 ≥ 90%，EER ≤ 10% |
| 团队规模 | 3 人 |
| 开发周期 | 7 天 |

## 4. 算法技术路线

### 识别流程

```
摄像头采图 → 手掌检测 → ROI裁剪(128×128) → Gabor滤波(6方向) → 二值编码 → 汉明距离匹配 → 阈值判决
```

### Gabor 特征原理

Gabor 滤波器模拟人类视觉皮层对纹理方向的响应。对掌纹图像进行 **6 个方向**（0°、30°、60°、90°、120°、150°）的 Gabor 卷积，对每个位置的响应取符号（正/负），编码为二进制串，即 **PalmCode**。

两幅掌纹的**汉明距离** = 编码中不同位的比例：

- 同一人：汉明距离通常 **< 0.35**
- 不同人：汉明距离通常 **> 0.45**

### 性能评估

遍历不同阈值，计算每个阈值下的 FAR（误接受率）和 FRR（误拒绝率），绘制 ROC 曲线，FAR = FRR 的交点即为 **EER（等错误率）**。EER 越小系统越好，课设目标 **EER < 10%**。

### 数据集准备

数据来源分两部分：**公开数据集**用于算法开发和性能评估，**自录数据**用于最终演示。

#### 公开数据集（用于算法调试）

| 数据集 | 人数/图像数 | 获取方式 | 链接 |
|--------|------------|----------|------|
| Kaggle Palm Recognition Dataset | 多人，含分类标注 | 免注册直接下载 | [链接](https://www.kaggle.com/datasets/saqibshoaibdz/palm-dataset) |
| Kaggle Hands and Palm Images | 多人 | 免注册直接下载 | [链接](https://www.kaggle.com/datasets/shyambhu/hands-and-palm-images-dataset) |
| PolyU-IITD Contactless v3.0 | 600+ 人，12000 张 | 需提交申请，约 45 天审批 | [链接](https://www4.comp.polyu.edu.hk/~csajaykr/palmprint3.htm) |
| IITD Touchless Palmprint | 230 人，各 7 张 | 邮件申请，时间不定 | [链接](https://www4.comp.polyu.edu.hk/~csajaykr/IITD/Database_Palm.htm) |

> **课设建议**：PolyU / IITD 审批周期太长（45天+），一周内来不及。优先用 Kaggle 数据集在 PC 上跑通算法、调好参数；PolyU 可以在报告里作为"标准基准数据集"引用，说明算法来源即可，不必实际下载。

#### 自录数据（用于演示）

最终答辩必须用自己的手实时识别，公开数据集无法替代这一步。采集方法：

- 采集人数：至少 3 人（建议 5 人，EER 评估更准）
- 每人每只手：30 张，包含不同角度、微偏移、不同光照
- 注册入库：每人取其中 5 张均值特征作为模板
- 测试集：剩余图像用于 FAR/FRR 评估

### GPIO 接线

```
树莓派 GPIO17 (Pin 11) → 继电器 IN
树莓派 GND    (Pin 6)  → 继电器 GND
树莓派 5V     (Pin 2)  → 继电器 VCC

继电器 COM → 12V 电源正极
继电器 NC  → 电磁门锁正极   (NC=常闭，断电开锁)
电磁门锁负极 → 12V 电源负极

GPIO23 (Pin 16) → 绿灯 → 330Ω → GND
GPIO24 (Pin 18) → 红灯 → 330Ω → GND
GPIO25 (Pin 22) → 蜂鸣器 → GND
```

> **注意**：门锁使用 12V 独立电源，不要从树莓派取电。继电器做强弱电隔离。

## 10. 风险与应对

| 风险 | 概率 | 应对方案 |
|------|------|----------|
| 电磁锁到货慢 | 中 | Day 1 立即下单；备选：LED 模拟开锁 |
| ROI 裁剪不稳定 | 中 | 演示台加手掌定位槽（卡纸引导手放固定位置） |
| 普通光下识别率差 | 低 | NIR 补光灯解决；备选：均匀白光补光 |
| 树莓派运算慢 | 低 | Gabor 用 NumPy 向量化；若仍慢缩小 ROI 至 64×64 |
| 代码接口不对齐 | 中 | Day 2 统一定义接口文档（ROI格式、特征维度、API路由） |
